# Technical Due Diligence Workflow

This package is the skeleton for a future VC-oriented technical due-diligence workflow.

The planned workflow will collect public product claims, docs/API/changelog/customer evidence,
competitor capabilities, factual release activity, unresolved evidence gaps, and code-review
follow-up questions.

The workflow is registered as `technical_due_diligence`. `workflow.py` orchestrates intake planning,
claim extraction, evidence research, competitor mapping, technical substance review, gap research,
and final memo generation.
